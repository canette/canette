# Security Policy for Canette

Version: 1.0

## Preface

Canette is a deployment tool and needs to have production access which makes
security a very important topic. The Canette team takes security very
seriously and is continuously working on improving it.

## Reporting a Vulnerability

If you find a security related bug in Canette, we kindly ask you for responsible
disclosure and for giving us appropriate time to react, analyze and develop a
fix to mitigate the found security vulnerability.

We will do our best to react quickly on your inquiry, and to coordinate a fix
and disclosure with you. As a small open-source team, there may be delays,
but we will make every effort to investigate and address valid reports.

We will publish security advisories using the
[GitHub Security Advisories](https://github.com/canette/canette/security/advisories)
feature to keep our community well-informed, and will credit you for your
findings (unless you prefer to stay anonymous).

To report a vulnerability to the Canette team, us the GitHub security advisory submission form: https://github.com/canette/canette/security/advisories/new

## Securing your Canette Installation

See the [documentation](https://canette.dev/docs/) for recommendations and additional information about running Canette in productions,
and the associated risks.
