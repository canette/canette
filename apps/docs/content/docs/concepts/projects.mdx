A **project** is an organisational unit that groups related apps. Projects belong to a [team](/docs/concepts/teams) — access to a project is granted by being a member of its team.

<Callout type="info">
  This page is a work in progress.
</Callout>

## What a project contains

- One or more **apps** (each app maps to a Git repository or container image)

## Access control

Access to a project is inherited entirely from team membership. All members of the owning team have full access to every project and app in that team. There are no per-project roles.

See [Teams](/docs/concepts/teams) for details on how to manage access.
